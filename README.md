# Autoware Control Log Monitor

Offline monitoring tool for vehicle control log data generated by Autoware control nodes. Given recorded runs, the tool assesses whether the vehicle's actual control behaviour satisfies formal monitoring conditions derived from a kinematic bicycle model.

## Requirements

- Python 3
- pandas
- numpy
- matplotlib

## Input data

Each run directory must contain two CSV files (no header row):

| File | Columns |
|------|---------|
| `control_log.csv` | `time, sta, v, a` (timestamp, steering angle [rad], velocity [m/s], acceleration [m/s²]) |
| `marker_log.csv` | `time, px, py, yaw, wx, wy, wv` (timestamp, vehicle position x/y [m], heading [rad], waypoint x/y [m], waypoint velocity [m/s]) |

## Usage

```bash
# Single-run mode with default settings (data directory: examples/1, window size: 100)
python3 main.py

# Single-run mode on a specific directory
python3 main.py --data-dir examples/0

# Single-run mode with custom window size
python3 main.py --data-dir examples/0 --window 200

# Single-run mode with debug plot (local waypoint geometry)
python3 main.py --data-dir examples/0 --debug

# Plot only specific rows (0-based indices)
python3 main.py --data-dir examples/0 --rows 0 9 16

# Batch mode: process all subdirectories under a parent directory
python3 main.py --batch --data-dir examples
```

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--data-dir DIR` | `examples/1` | Log directory (single-run) or parent directory (batch) |
| `--batch` | — | Process all subdirectories under `--data-dir` |
| `--workers N` | — | Number of parallel workers in batch mode (default: number of CPUs) |
| `--window N` | `100` | Sliding window size in rows |
| `--debug` | — | Show additional debug plot (local waypoint geometry) |
| `--rows N...` | — | Plot only specified row indices (0-based, single-run only) |
| `--wb F` | `2.79` | Vehicle wheelbase [m] |
| `--aa F` | `1.0` | Upper acceleration bound [m/s²] |
| `--bb F` | `1.0` | Lower (deceleration) bound [m/s²] |
| `--eps F` | `0.5` | Spatial tolerance [m] |
| `--eps-v F` | `0.2` | Velocity tolerance [m/s] |
| `--th F` | `0.001` | Time step [s] |

## Output

### Single-run mode

A per-window summary is printed to stdout:

```
i: (ann1&ann2&speed1&speed2)&go1&go_h&go_l; <init>; <reached>; dist=<d>
```

- Each of the seven accumulator values is `1` (all rows in the window passed) or `0` (at least one row failed).
- `<init>` — initial-condition result: `Iok` (all satisfied) or `Ing<r>` where `<r>` is a bitmask of violated conditions (ann1=1, ann2=2, speed1=4, speed2=8, go_h=16, go_l=32).
- `<reached>` — waypoint-reached result: `Rok` (both conditions met) or `Rng<r>` where `<r>` encodes which condition was not met when the window ended (eps-ball=1, velocity=2).
- `dist` — displacement of the waypoint between the first two rows of the window (used for the skip heuristic).

Three plots are displayed side by side:

- **Global trajectory** — vehicle positions (filled circles), waypoints (hollow circles), vehicle-to-waypoint lines, and steering direction arrows.
- **Monitoring result** — one dot per row in the window, color-coded green (pass) / red (fail).
- **Local waypoint geometry** (with `--debug`) — waypoint positions in the vehicle-local frame.

### Batch mode

Per-run and overall pass/fail counts are printed to stdout:

```
<subdir>: <valid_count>/<total_count>
...
total: <valid_total>/<total>
```

## Monitoring conditions

For each sliding window the tool evaluates the following conditions row by row and accumulates a conjunctive result:

| Condition | Formula |
|-----------|---------|
| `ann1` | `\|κ\| · ε ≤ 1` |
| `ann2` | `κ · \|(wx²+wy²−ε²)/2 − wy\| < ε` |
| `speed1` | `0 ≤ vl < vh` |
| `speed2` | `aa·th ≤ vh−vl` and `bb·th ≤ vh−vl` |
| `go1` | `−bb ≤ acc ≤ aa` and `vel+acc·th ≥ 0` |
| `go_h` | `vel ≤ vh` and `vel+acc·th ≤ vh`, or `(1+\|κ\|·ε)²·(vel·th + (acc/2)·th² + ((vel+acc·th)²−vh²)/(2·bb)) + ε ≤ wx` |
| `go_l` | `vl ≤ vel` and `vl ≤ vel+acc·th`, or `(1+\|κ\|·ε)²·(vel·th + (acc/2)·th² + (vl²−(vel+acc·th)²)/(2·aa)) + ε ≤ wx` |

where `κ` is curvature derived from the steering angle, `wx`/`wy` are the waypoint coordinates in the vehicle-local frame, and `vl = wv − eps_v`, `vh = wv + eps_v` are the velocity bounds.
