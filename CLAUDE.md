# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

This repository manages a script that implements an off-line monitoring method for the log data related to vehicle control generated by the Autoware control nodes.

## Documents

All documents should be written in English (although conversations are conducted in another language).

1. Persistent documents are placed under `docs/`: 
    - Define *what* the scripts should do and *how* they should be developed.
    - *`requirements.md`*
        - Objective of the project
        - Main functions
        - Basic process of the implemented method
        - Usage examples
    - *`repository-structure.md`*
        - Structure of the folders and files
        - File placement rules

2. Documents for each work unit, named `.steering/[YYYY-MM-DD]_[name-of-the-unit].md`: 
    - Specify the requirements in the unit.
    - Explain the basic design.
    - List the concrete tasks and their progress.
    - *Important*: Creation and modification of the files require approval.

## Development process

1. Prepare persistent documents.
    - `docs/requirements.md`
    - `docs/repository-structure.md`
2. Create a steering file for the initial implementation.
    - `.steering/[YYYY-MM-DD]_initial-implementation.md`
3. Start the development based on the task list in the steering file.
4. Verify the implementation by writing and running pytest tests.
