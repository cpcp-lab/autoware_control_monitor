# Requirements

## Objective

Implement an offline monitoring tool for vehicle control log data generated by Autoware control nodes. Given a set of recorded runs, the tool assesses whether the vehicle's actual control behaviour satisfies formal monitoring conditions derived from a kinematic bicycle model, and aggregates the results across runs to evaluate controller accuracy and detect anomalies.

### Current scope

Single-run analysis: load one log directory, evaluate monitoring conditions per sliding window, and visualize the results.

## Main functions

### Overall pipeline

1. **Multi-run loading** — scan a collection of log sub-directories `1/` to `n/`.
2. **Per-run monitoring** — apply single-run analysis to each directory `i/`.
3. **Result aggregation and evaluation of the controller** — aggregate the results and identify the worst error across runs.

### Single-run analysis (current implementation)

1. **Log loading** — read `i/control_log.csv` and `i/marker_log.csv` from a user-specified directory `i/`.
2. **Time synchronization** — align the two logs by timestamp so that each marker sample carries the most recent control values.
3. **Windowed sampling** — divide the marker log into fixed-size sliding windows and select one representative row per window.
4. **Monitoring evaluation** — for each window, evaluate annotation, feasibility, and go conditions row by row and accumulate conjunctive results.
5. **Visualization** — render three subplots: global trajectory, local waypoint geometry, and per-waypoint monitoring outcome.
6. **Result reporting** — print a one-line summary per window to stdout.

## Basic process of the implemented method

```
load control_log.csv, marker_log.csv
  ↓
merge_asof: attach nearest past control record to each marker row
  ↓
sample every `window` rows → m_subset
  ↓
for each sampled row (window start):
    determine idx_end: first row where the waypoint has passed behind the vehicle
    for each row in [idx, idx_end]:
        compute waypoint offset in vehicle-local frame (wx1, wy1)
        compute curvature κ from steering angle
        compute running-average acceleration
        evaluate:
            ann1: |κ| · ε ≤ 1
            ann2: κ · |(wx1²+wy1²−ε²)/2 − wy1| < ε
            feas1: wx1 > 0  and  0 ≤ vl < vh
            feas2: aa·th ≤ vh−vl  and  bb·th ≤ vh−vl
            go1:  −bb ≤ acc ≤ aa  and  vel+acc·th ≥ 0
            go_h: vel ≤ vh  (or braking distance condition)
            go_l: vl ≤ vel  (or acceleration distance condition)
        accumulate conjunctive result
    print summary:  i: (ann1&ann2&feas1&feas2)&go1&go_h&go_l
```

## Usage examples

```bash
# Run with default settings (data directory: 1, window: 100)
python3 main.py

# Specify a data directory
python3 main.py examples/0

# Specify window size
python3 main.py --window 200 examples/0
```

## Visualization

Three subplots are produced side by side.

**Plot 1 — Global trajectory:** vehicle positions (filled circles), waypoints (hollow circles), vehicle-to-waypoint lines, and steering direction arrows.

**Plot 2 — Monitoring result:** one dot per row in `[idx, idx_end]`, color-coded green (pass) / red (fail) based on `feas & go`. The origin marks the reference point.

Per-window summary printed to stdout:
```
i: (ann1&ann2&feas1&feas2)&go1&go_h&go_l
```
where each value is 1 (all rows passed) or 0 (at least one row failed).

## Parameters

| Variable | Default | Meaning |
|----------|---------|---------|
| `wb` | 2.79 | Vehicle wheelbase [m] |
| `aa` | 1.0 | Upper acceleration bound [m/s²] |
| `bb` | 0.5 | Lower (deceleration) bound [m/s²] |
| `eps` | 0.1 | Spatial tolerance [m] |
| `eps_v` | 0.001 | Velocity tolerance [m/s] |
| `th` | 0.001 | Time step [s] |
